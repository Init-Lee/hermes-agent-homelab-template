## Rules for Agents

Agents should treat this directory as their default working boundary.

Agents may read/write inside `/srv/agent-workspace`.

Agents must not modify the following without explicit human confirmation:

- `/dev/*`
- `/boot`
- `/etc/fstab`
- `/etc/ssh/*`
- `/etc/systemd/*`
- disk partitions
- RAID arrays
- mount points
- Windows NTFS partitions
- firewall rules
- SSH configuration
- API keys or secret files
- system packages
- files outside `/srv/agent-workspace` and `~/.hermes`

## Disk Safety Notes

Before any RAID, mount, format, or partition operation, the agent must show:

```bash
lsblk -o NAME,SIZE,TYPE,FSTYPE,MOUNTPOINT,MODEL
```

and wait for human confirmation.
